# Security Policy

## Supported versions

The project is pre-1.0. Security fixes target the latest commit on `main`.

## Reporting a vulnerability

Please open a private security advisory on GitHub if available. If that is not available, open an issue with minimal public detail and request a maintainer contact path.

Do not include live secrets, credentials, private repository data, or exploit payloads in public issues.

## Security-sensitive areas

- File generation and overwrite behavior
- GitHub Action execution
- Path handling
- Future network or API integrations
- Future AI-generated PR review and issue triage output
